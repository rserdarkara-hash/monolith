# Specification: Scientific Analytics Engine (Monolith v0.8.0)

## Overview
Develop a new "Scientific Analytics Engine" module that operates as a new primary tab ("Tab 5") within the Monolith Spatial Analysis Dashboard. This module integrates a comprehensive suite for descriptive statistical analysis, correlation mapping, and PCA, utilizing both `ggplot2` and `plotly` to provide deep insights into raw and interpolated soil properties. Crucially, the application (currently `monolith_0.7.0.R`) and its helper files must be backed up before initiating development on `monolith_0.8.0.R`.

## Functional Requirements

### 1. General Architecture & UI
- **New Tab Integration:** Add a new main tab (e.g., "Tab 5: Scientific Analytics") dedicated to this engine.
- **Plot Expandability:** Implement an "Expandable Thumbnails" UI. Plots in the main view will act as thumbnails (using `ggplot2`). Expanding a plot will offer two modes: a full-scale `ggplot2` rendering or an interactive `plotly` version (where relevant, e.g., for hovering over data points).
- **Tabular Data Sync:** Every visualization must feature an "Export Tabular Data" button that extracts the specific binned, transformed, or filtered data generated by the plot (e.g., via `ggplot_build(p)$data`).

### 2. Descriptive Suite
- Implement a distribution-focused UI with: Sinaplots, Histograms, Violin plots, Box & Whiskers, Line plots, Ridge Plots, ECDF, QQ-plots, Density Heatmaps, Joyplots, Parallel Coordinate plots, and Radar charts.
- Include XYZ Surface plots supporting 5 fit types: Linear, Loess, Thin Plate Splines, Polynomial (2nd degree), GAM.
- **"Ghosting" Feature (Local vs. Global):** When a locality is selected, Histograms/Density Plots, Box/Violin Plots, Scatterplots, and ECDFs must display the locality distribution in full color overlaid on a light gray "shadow" of the global/total distribution.

### 3. Correlation Analysis
- Create a dedicated panel for correlation analysis supporting both global and per-locality contexts.
- Features: Hierarchical Clustering Heatmaps (with significance), Correlation Networks, Partial Correlation Plots, Correlograms, and Lagged Correlation plots.

### 4. PCA Module
- High-dimensional analysis suite: Scree plots (Kaiser criterion), Biplots, Loadings plots.
- Advanced metrics: Contribution plots, Variable Importance maps, Cumulative Variance plots, 3D Interactive Biplots, Mahalanobis Distance/Outlier plots.
- **Automated Collinearity Filter:** Before PCA execution, check for near-zero variance or perfect collinearity ($r > 0.95$). If detected, warn the user and present the data, giving them the option to ignore the warning and continue, or manually interfere (e.g., exclude variables).

### 5. Grouping & Discretization Logic
- Enable a multi-factor grouping system (up to 5 factors) with toggles for categorical vs. numeric treatment.
- Automatic discretization features:
  - Central Tendency (mean/median splits).
  - Quantiles (Tertiles, Quintiles).
  - User-Defined custom breakpoints.
- Provide searchable statistical tables nested directly below each plot.

## Non-Functional Requirements
- **Backup Strategy:** The very first step of the implementation must be to backup `monolith_0.7.0.R` and its helper files into the `backups/` directory before starting work on v0.8.0.
- **Tech Stack Compliance:** Maintain existing `ggplot2`, `patchwork`, and spatial libraries. Integrate `plotly` responsibly for the interactive expansions.

## Acceptance Criteria
- [ ] A new main tab for the "Scientific Analytics Engine" is functional.
- [ ] All required plot types in the Descriptive Suite are rendered correctly with proper axes control.
- [ ] The "Ghosting" feature correctly overlays locality data onto global background data for the specified plot types.
- [ ] Correlation network and heatmaps generate correctly based on selected variables.
- [ ] PCA module detects collinearity ($r > 0.95$) and provides the user with an interactive warning/resolution prompt before proceeding.
- [ ] Expanding a plot offers both `ggplot2` and `plotly` (where relevant) interactive views.
- [ ] "Export Tabular Data" correctly outputs the processed plot data (not just raw data).
